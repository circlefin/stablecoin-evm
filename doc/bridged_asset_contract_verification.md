# Bridged Asset Contract Verification

## Data collection from contract deployers

The following contracts must be verified to ensure Bridge to Native compliance.

- `FiatTokenProxy`
- `FiatTokenV2_2` or latest implementation
- `SignatureChecker`

Before collecting data from the deployers, first ensure that partners have
deployed tokens per [Circle's instructions](./bridged_USDC_standard.md). The
following verification would only work for contracts deployed with the exact
same file structure, optimizer run configuration, and evm version settings.

The deployer of these contracts should supply the following to Circle for
verification.

- A public RPC endpoint for the chain where contracts have been deployed.
- Addresses of all 3 deployed contracts
- The contract creation transaction hash of all 3 deployed contracts (usually
  available from the block explorer)
- Compiled metadata file for all 3 of the deployed contracts. These can be
  generated by the deployer following
  [these instructions](https://docs.soliditylang.org/en/latest/metadata.html).
  Detailed instructions on extracting metadata files based on different
  development frame work can be found [here](./metadata_extraction.md).

## Repo Setup

Open a new terminal window and run the following command:

```sh
git clone git@github.com:crcl-main/stablecoin-evm-private.git "stablecoin-evm-private" && \
cd "stablecoin-evm-private" && \
nvm use && \
yarn install && \
yarn compile
```

## Contract Verification

1. Copy supplied metadata files to under `stablecoin-evm-private` directory.

2. Verify proxy contract

```sh
MAINNET_RPC_URL=<PUBLIC_RPC_ENDPOINT> \
yarn hardhat verifyOnChainBytecode \
--contract-name FiatTokenProxy \
--contract-address <FIAT_TOKEN_PROXY_CONTRACT_ADDRESS> \
--metadata-file-path <FIAT_TOKEN_PROXY_METADATA_FILE_PATH> \
--contract-creation-tx-hash <FIAT_TOKEN_V2_2_CREATION_TX_HASH> \
--network mainnet
```

3. Verify implementation contract

```sh
MAINNET_RPC_URL=<PUBLIC_RPC_ENDPOINT> \
yarn hardhat verifyOnChainBytecode \
--contract-name FiatTokenV2_2 \
--contract-address <FIAT_TOKEN_V2_2_CONTRACT_ADDRESS> \
--library-name SignatureChecker \
--library-address <SIGNATURE_CHECKER_CONTRACT_ADDRESS> \
--metadata-file-path <FIAT_TOKEN_V2_2_METADATA_FILE_PATH> \
--contract-creation-tx-hash <FIAT_TOKEN_V2_2_CREATION_TX_HASH> \
--network mainnet
```

4. Verify signature checker library contract

```sh
MAINNET_RPC_URL=<PUBLIC_RPC_ENDPOINT> \
yarn hardhat verifyOnChainBytecode \
--contract-name SignatureChecker \
--contract-address <SIGNATURE_CHECKER_CONTRACT_ADDRESS> \
--is-library true \
--metadata-file-path <SIGNATURE_CHECKER_METADATA_FILE_PATH> \
--contract-creation-tx-hash <SIGNATURE_CHECKER_CREATION_TX_HASH> \
--network mainnet
```

If all steps above return passing results, we can consider the contracts
verified. If any of the validation fails, please contact the Circle eng team for
additional validation.
